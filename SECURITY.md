# Security Policy

## Responsible Use

OyShell is a security research tool intended for **authorized testing only**. 

### Permitted Uses
- Penetration testing with written authorization
- Security research in controlled environments
- Educational purposes in lab settings
- Testing your own infrastructure

### Prohibited Uses
- Attacking systems without explicit permission
- Disrupting services you don't own or have permission to test
- Any activity violating local, state, or federal laws
- Harassment or malicious targeting of individuals

## Reporting Vulnerabilities

If you discover a security vulnerability in OyShell itself:

1. **Do not** open a public issue
2. Email: security@oystra.ai
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

We will respond within 72 hours and work with you on disclosure.
